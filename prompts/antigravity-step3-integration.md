# Antigravity Prompt — Step 3: Page Integration
> **Model:** Claude Sonnet 4.5 (in Antigravity — set model to Claude Sonnet 4.5 before running)
> **Mode:** Agent (not Planning — let it run autonomously)
> **When to use:** After Gemini Flash generates services.html or projects.html and you've dropped the file into the workspace
> **What it does:** Integrates the Gemini output into the site, ensures consistency with existing pages, and visually verifies in the browser.

---

## ✅ Prompt Template

```
You are an autonomous frontend integration agent. Your job is to take a newly generated HTML page and fully integrate it into an existing multi-page website, then verify the result in the browser.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
TASK
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Integrate [services.html / projects.html] into the EURL ETB Achouri Toufik website so it is indistinguishable in design and behavior from the existing pages (index.html, about.html, contact.html).

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
PROJECT FILE STRUCTURE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- index.html       → main reference page (navbar + footer source of truth)
- about.html       → secondary reference
- contact.html     → secondary reference
- style.css        → global styles
- layout.css       → layout styles
- script.js        → shared JavaScript (GSAP, hamburger, scroll-spy)
- [target].html    → the new file to integrate (generated by Gemini)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
STEP 1 — AUDIT THE NEW FILE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Open [target].html and index.html side by side. Check for these differences and fix every one:

NAVBAR:
□ Is the navbar identical in structure to index.html? (both topbar + mainbar rows)
□ Is the correct nav link marked active? (add class="active" to the right <a> tag)
□ Is the hamburger + mobile drawer present and functional?
□ Does the logo match index.html exactly?

FOOTER:
□ Is the footer identical in structure and content to index.html?
□ Are all footer links pointing to the correct .html files?
□ Is the newsletter input and subscribe button present?

CSS / TAILWIND:
□ Is the Tailwind CDN script present in <head>?
□ Is the tailwind.config block present with all custom colors and fonts?
□ Are all custom CSS classes from index.html present in a <style> tag?
  Required classes: .blueprint-subtle, .nav-link, .nav-link.active, .hamburger,
  .nav-drawer, .fnav-link, .footer-topline, .service-card (or .project-card),
  .project-overlay (if projects page)

FONTS:
□ Is the Google Fonts link present for Barlow, Barlow Condensed, and Lato?

SCRIPTS:
□ Are GSAP and ScrollTrigger CDN scripts present before the closing </body>?
□ Is all JS inside window.addEventListener('load', () => { ... })?
□ Does the page include: hamburger toggle, scroll animations, and any page-specific JS?

LINKS:
□ Do all internal links (nav, footer, CTAs) point to correct .html filenames?
□ Are there any broken href="#" placeholders that should link to real pages?

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
STEP 2 — FIX ALL ISSUES FOUND
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
For every issue found in Step 1:
- Edit [target].html directly
- Copy the exact navbar HTML from index.html if the navbar differs significantly
- Copy the exact footer HTML from index.html if the footer differs significantly
- Do not rewrite sections that are correct — only fix what's broken
- Add a comment above each fix: <!-- INTEGRATED: reason -->

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
STEP 3 — OPEN IN BROWSER AND VERIFY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Open [target].html in the browser. Visually confirm:

□ Page loads without console errors
□ Navbar appears correctly at the top (navy bg, gold bottom border, logo + links visible)
□ Active nav link has the gold underline on the correct item
□ Page hero section is visible below the navbar (not hidden behind it)
□ All sections render with correct colors (navy #1e243c, gold #c19f5d, light-bg #f4f4f2)
□ No white flashes or unstyled content on load
□ Footer renders correctly at the bottom
□ GSAP animations trigger on scroll (sections fade in as you scroll down)

If any visual issue is found, fix it and re-verify in the browser before finishing.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
STEP 4 — CROSS-PAGE NAVIGATION CHECK
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- Open index.html in the browser
- Click the nav link that goes to [target].html
- Confirm the page loads correctly and the active nav link updates
- Click "Home" from [target].html and confirm index.html loads correctly
- Repeat for "Contact" link

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
COMPLETION CRITERIA
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Do not mark this task complete until ALL of the following are true:
✓ All checklist items in Step 1 are resolved
✓ Browser shows no console errors
✓ Page is visually consistent with index.html
✓ Navigation between pages works in both directions
✓ GSAP scroll animations are working

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
OUTPUT REPORT
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
When complete, output a short summary:
- List every fix applied and why
- Confirm browser verification passed
- Flag anything that needs human review
```

---

## 📥 How to use this in Antigravity
1. Make sure Gemini Flash has already generated the file and you've saved it to the workspace
2. Open Antigravity → Agent Manager → start a new conversation
3. Set model to **Claude Sonnet 4.5** (not Gemini)
4. Set mode to **Agent** (not Planning)
5. Type `Read @prompts/README.md first and follow all rules in it before doing anything.` at the top
6. Paste the prompt below, replacing `[target].html` with the actual filename
7. Let it run — it will edit files, open the browser, and verify autonomously
8. Paste Claude's output report back into claude.ai for Step 5 review
